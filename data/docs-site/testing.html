2. Authenticated mutating tests

Mint a session first; then exercise the full enter → submit → settle flow.

2.1 Mint a device session (Bash)

SERVER_KEY=defaultkey   # client-facing key
DEVICE_ID=$(uuidgen)

SESSION=$(curl -sS -X POST \
  "$NAKAMA/v2/account/authenticate/device?create=true&username=tester_$DEVICE_ID" \
  -u "$SERVER_KEY:" \
  -H 'Content-Type: application/json' \
  --data "{\"id\":\"$DEVICE_ID\"}")

TOKEN=$(echo $SESSION | jq -r .token)
USER_ID=$(echo $SESSION | jq -r .user_id)
echo "Logged in as $USER_ID"

2.2 Pre-enroll for the founder cohort

curl -sS -X POST "$NAKAMA/v2/rpc/tournament_pre_enroll?unwrap" \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  --data '"{\"slug\":\"gk-royale-daily\"}"' | jq

# Expect: {"ok":true,"founder":true,"total_enrolled":N+1}
# Or on second call: {"ok":true,"idempotent":true,"founder":true}

2.3 Try to enter (will fail — slug is PRE_ENROLL not OPEN)

IDEMP=$(uuidgen)

curl -sS -X POST "$NAKAMA/v2/rpc/tournament_enter?unwrap" \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  --data "{\"slug\":\"gk-royale-daily\",\"idempotency_key\":\"$IDEMP\"}" | jq

# Expect: {"ok":false,"error":"tournament_not_open","status":"PRE_ENROLL"}

This is the correct, expected behavior before public launch on Jul 1, 2026. After that date, the same call returns {"ok":true,"entry_id":"...","pot_bc":...}.

3. Day-of-launch flow (post Jul 1, 2026)

Run this on a fresh test account once a slug is OPEN.

3.1 Enter, submit, expect rank

# 1. Enter
curl -sS -X POST "$NAKAMA/v2/rpc/tournament_enter?unwrap" \
  -H "Authorization: Bearer $TOKEN" \
  --data "{\"slug\":\"gk-royale-daily\",\"idempotency_key\":\"$(uuidgen)\"}"

# Expect debit of 50 BC + entry_id

# 2. Fetch a pack
PACK=$(curl -sS -X POST "$NAKAMA/v2/rpc/tournament_content_get_pack?unwrap" \
  -H "Authorization: Bearer $TOKEN" \
  --data '"{\"slug\":\"gk-royale-daily\",\"pack_index\":0}"')

PACK_ID=$(echo $PACK | jq -r .data.pack_id)

# 3. Submit answers (all correct, realistic latency)
curl -sS -X POST "$NAKAMA/v2/rpc/tournament_submit_pack_result?unwrap" \
  -H "Authorization: Bearer $TOKEN" \
  --data "{\"slug\":\"gk-royale-daily\",\"pack_id\":\"$PACK_ID\",\"answers\":[1,2,3,4,5],\"client_latency_ms\":850}"

# Expect: {"ok":true,"score":N,"your_rank":N}

3.2 Watch pot tick on the leaderboard

curl -sS -X POST "$NAKAMA/v2/rpc/tournament_leaderboard_top?unwrap" \
  -H "Authorization: Bearer $TOKEN" \
  --data '"{\"slug\":\"gk-royale-daily\",\"limit\":10}"' | jq

# Expect: {"ok":true,"data":{"rows":[{"rank":1,"username":"...","score":...},...]}}

3.3 Settle a test tournament (admin/cron-only)

# Service-only — requires the admin bearer.
SERVICE_TOKEN="$TOURNAMENT_SERVICE_TOKEN"

curl -sS -X POST "$NAKAMA/v2/rpc/tournament_settle?unwrap" \
  -H "Authorization: Bearer $SERVICE_TOKEN" \
  --data '"{\"slug\":\"test-tournament\"}"' | jq

# Expect: {"ok":true,"payouts_distributed":N,"rake_bc":N,"ledger_drift":0}

3.4 Claim a certificate

curl -sS -X POST "$NAKAMA/v2/rpc/tournament_claim_certificate?unwrap" \
  -H "Authorization: Bearer $TOKEN" \
  --data '"{\"slug\":\"test-tournament\"}"' | jq

# Expect: {"ok":true,"url":"https://intelli-verse-x-media.s3...certificates/test-tournament/USER.png","cert_id":"..."}
