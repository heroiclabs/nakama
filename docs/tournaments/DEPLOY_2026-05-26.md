## Cron requirement after deploy

The opportunistic tick inside `tournament_list` and `tournament_get` runs at
most once per 60 seconds and only when read-traffic comes in. For tournaments
to advance even when nobody is browsing, **add an external scheduler that
calls `tournament_cron_tick` every minute**:

```yaml
# Kubernetes CronJob example
apiVersion: batch/v1
kind: CronJob
metadata:
  name: tournament-cron-tick
spec:
  schedule: "* * * * *"
  jobTemplate:
    spec:
      template:
        spec:
          containers:
            - name: tick
              image: curlimages/curl:latest
              args:
                - -fsS
                - -X
                - POST
                - https://nakama.prod.intelli-verse-x.ai/v2/rpc/tournament_cron_tick
                - -H
                - "Authorization: Bearer $(SERVICE_TOKEN)"
                - -d
                - '{"service_token":"$(TOURNAMENT_SERVICE_TOKEN)"}'
          restartPolicy: OnFailure
```

The opportunistic tick is enough for staging and the first week post-launch
(plenty of read traffic), but the external CronJob is recommended for
overnight quiet hours so daily roll-forward fires reliably at 00:00 UTC.
