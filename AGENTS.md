# Full stack start

docker compose build nakama && docker compose up -d --force-recreate nakama

# Stream logs
docker compose logs -f nakama

# Open DB shell

## 🔑 Environment Variables

All secrets live in `.env` (gitignored). Pass to JS runtime via `--runtime.env` in docker-compose.yml entrypoint.
Access in JS as `ctx.env['KEY_NAME']`.

**Never commit `.env` or any credentials.**

Key vars: `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `ADMIN_USERNAME`, `ADMIN_PASSWORD_HASH`,
`NAKAMA_WEBHOOK_SECRET`, `APPLE_KEY_ID`, `APPLE_ISSUER_ID`, `APPLE_PRIVATE_KEY`,
`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `DEFAULT_GAME_ID`.

## 📜 Always-Active Rules

1. **SKILL_BOOK first** — match task → load only listed skill (max 2 skills per task)
2. **NEVER hand-edit `index.js`** — 100% auto-generated by postbuild.js
3. **NEVER use Node built-ins** — no `require('fs')`, `require('http')`, etc. in JS/TS
4. **NEVER use global vars for state** — Goja VM pool; each call is isolated
5. **TypeScript target MUST be `es5`** — never change in tsconfig
6. **All RPC functions in global scope** — no `var fn = initializer.registerRpc(...)` pattern
7. **Env vars for JS MUST be in RUNTIME_ENV_KEYS** in docker-compose.yml entrypoint
8. **Build + restart after every change** — modules loaded at server startup only
9. **Minimum viable change** — do only what's requested, confirm if scope seems larger
10. **Ask before expanding scope** — don't add modules/RPCs beyond what's requested

## 🚫 Forbidden

| Action | Why |
|--------|-----|
| Edit `data/modules/index.js` directly | Auto-generated; next build overwrites it |
| Use `require('fs')`, `require('http')`, etc. | Goja sandbox — no Node APIs |
| Store state in module-level vars | Goja VM pool — state resets between calls |
| Commit `.env` or credentials | Security |
| Use `target: "es2016+"` in tsconfig | Goja only supports ES5 |
| Direct `var fn = rpc(...)` as registerRpc arg | Nakama AST walker won't detect it |
| Modify Go source without rebuilding image | Plugin ABI mismatch |

### Read-Only / Do Not Touch
```
vendor/                   ← Go vendor tree (never modify)
.github/                  ← CI config
go.mod / go.sum           ← pinned deps
```

## 🧠 Learned Facts

- **postbuild.js v2** rewrites `registerRpc("id", fn)` → `__rpc_id = fn` + generates a wrapper `InitModule` with direct calls. This is mandatory because Nakama's `getRegisteredFnIdentifier` AST walker only sees top-level statements.
- **`ctx.env['KEY']`** only works for vars listed in `RUNTIME_ENV_KEYS` in the docker-compose entrypoint, NOT for vars in the `environment:` block alone.
- **CockroachDB v24.1** (not Postgres) — SQL dialect is mostly Postgres-compatible but some differences exist.
- **Go plugin ABI**: `analytics_metrics.so` must be built against exact Nakama version (3.35.0). ABI mismatch = panic at startup.
- **`DEFAULT_GAME_ID`** defaults to QuizVerse UUID (`126bf539-dae2-4bcf-964d-316c0fa1f92b`).
- **Nakama Console** default creds: `admin` / `password` (change via `ADMIN_USERNAME`/`ADMIN_PASSWORD_HASH` env).
- **Heroic Labs LLM docs**: every doc page has a `/llm.md` variant (e.g., `heroiclabs.com/docs/.../typescript-runtime/llm.md`) — use these for clean context.
- **RPC naming**: `quizverse_*`, `lasttolive_*`, `ai_pipeline_*`, `analytics_*`, plain (shared platform).
- **nakama-common package**: `npm install "https://github.com/heroiclabs/nakama-common#<tag>"` — must match Nakama binary version.
- **`data/modules/src/`**: TypeScript source root. All new TS goes here.
